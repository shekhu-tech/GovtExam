8. Prompting Best Practices üìà

To get the best from Mistral's API models, it's important to use clear instructions and leverage features like system prompts.

üìö Key Principles:

  1. Use a System Prompt

    Set the model's behavior, persona, and constraints at the beginning of the conversation using the `system` role for more consistent outputs.

  2. Be Specific

    Provide clear, detailed instructions. The more specific you are about the desired tone, format, and content, the better the result.

üîÑ Advanced Techniques:

  • Function Calling: Use the model's function calling capabilities to connect it to external tools and APIs, enabling more dynamic applications.
  • Few-Shot Prompting: Provide a few examples within your prompt to guide the model towards the desired output style and format.

Stage 1: Pre-training

The model is first pre-trained on trillions of tokens of text and code from the public internet. This allows the model to learn grammar, facts, reasoning patterns, and the structure of language on a massive scale, forming a broad base of knowledge.

Stage 2: Fine-Tuning

After pre-training, the model is fine-tuned on high-quality, instruction-based datasets. This stage uses techniques like Supervised Fine-Tuning (SFT) and Direct Preference Optimization (DPO) to teach the model how to follow instructions, be a helpful conversationalist, and align with safety protocols.

Output: Helpful Response

When a user sends a prompt, the fully trained model generates a response that is contextually relevant, follows the given instructions, and adheres to the safety guardrails established during the fine-tuning process, resulting in a helpful and reliable output.
